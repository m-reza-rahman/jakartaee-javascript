JavaScript/HTML5 Clients with Jakarta EE 10

This application demonstrates how Jakarta EE 10 can be an extremely effective back-end for JavaScript/HTML5 clients. The application shows the use of APIs like WebSocket, JSON Processing, REST, CDI, Validation, and Persistence. React is used as the JavaScript front-end framework. However, the choice of React is somewhat arbitrary and not central to the application. The concepts and back-end could also be utilized by Angular or Vue.

There are three main parts of the application:

  • A to do interface that utilizes REST.
  • A chat interface that utilizes WebSocket.
  • A live JVM memory monitoring interface that utilizes SSE.

Note that all three interfaces are secured using HTTP BASIC authentication. You can set the usernames and passwords in the SQL load scripts of the application source.

The back-end REST, SSE, and WebSocket APIs are documented here.
